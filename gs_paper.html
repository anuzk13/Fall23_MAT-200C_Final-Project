3D Gaussian Splatting for Real-Time Radiance Field Rendering

Intro
...

Note

At the core of their contribution is a new form of representation of 3D data for optimizing a differentiable rendering process, an explicit representation which is unstructured gaussians (kind of like a point cloud) instead of the implicit representation that NERFs used (SFD)

"Meshes and points are the most common 3D scene representations because they are explicit and are a good fit for fast GPU/CUDA-based rasterization. In contrast, recent Neural Radiance Field (NeRF) methods build on continuous scene representation" ... “While the continuous methods helps optimization, the stochastic sampling required for rendering is costly and can result in noise.” (Kerbl et al., 2023, p. 1)

  1. “We first introduce (anisotropic) 3D Gaussians as a flexible and expressive scene representation. We start with the same input as previous NeRF-like methods, i.e., cameras calibrated with Structure-from-Motion (SfM)” (Kerbl et al., 2023, p. 2)
    • An anisotropic object or pattern has properties that differ according to direction of measurement.
  2. “The second component of our method is optimization of the properties of the 3D Gaussians – 3D position, opacity 𝛼, anisotropic covariance, and spherical harmonic (SH) coefficients – interleaved with adaptive density control steps, where we add and occasionally remove 3D Gaussians during optimization” (Kerbl et al., 2023, p. 2)
  3. The third and final element of our method is our real-time rendering solution that uses fast GPU sorting algorithms and is inspired by tile-based rasterization” ([Kerbl et al., 2023, p. 2](3D Gaussian Splatting for Real-Time Radiance Field Rendering))

(Kerbl et al., 2023, p. 4)

  • Input
    • “a set of images of a static scene,
    • corresponding cameras calibrated by SfM which produces a sparse point cloud as a side effect.”
    • SfM is calculated using colcam
    1. create “3D Gaussians (Sec. 4), defined by:
    • a position (mean),
    • covariance matrix
    • opacity 𝛼”
    1. create a radiance field “via a sequence of optimization steps of 3D Gaussian parameters, i.e., position, covariance, 𝛼 and SH coefficients interleaved with operations for adaptive control of the Gaussian density.”
    • “The directional appearance component (color) of the radiance field is represented via spherical harmonics (SH), following standard practice”
    1. rasterization “our tile-based rasterizer (Sec. 6) that allows 𝛼-blending of anisotropic splats, respecting visibility order thanks to fast sorting.”

  • Requirements of a primitive:

    • “we need a primitive that inherits the properties of differentiable volumetric representations, (like SDF) while at the same time being unstructured (like points) and explicit to allow very fast rendering.” (Kerbl et al., 2023, p. 4)

  • Gaussians:

    -> Covariance Matrix, defines how the Gaussian blob is shaped in 3D space. It determines how wide or narrow the Gaussian is in each direction (x, y, z). It also defines the orientation of the Gaussian distribution by describing the relationship between the variables. For example, if the Gaussian is elongated in one direction, the covariance matrix will show a larger value for that direction.
    -> This is a matrix multiplication and results in a scalar value. The expression essentially measures how far the point is from the center of the Gaussian distribution, taking into account the shape and orientation of the distribution as described by . The farther away is, the larger this value becomes.
    -> This exponent determines the "height" of the Gaussian function at the point . Points closer to the center of the Gaussian will have a higher value (as the exponent approaches zero), while points further away will rapidly decrease towards zero, creating the bell shape.
