Locally deploying the GS paper
...

Instructions I followed to deploy the code published by the GS paper authors at

  • Most of the instructions by this video were very helpful but I had to eventually divert because the instructions there did not work for me.

Setup
...

I am working on a windows laptop with an NVIDIA GeForce RTX 3080 Laptop graphics card with 16 GB of memory

Requirements
...

  1. Install VS Code 2019
  2. Install CUDA Toolkit IMPORTANT- use 11.7
    • The original repo and youtube sugges to test with "11.8. Ensure you are not running 11.6 or 12+."
    • I, however found that the version that worked is 11.7 and [this issue in github helped me find the correct steps to run the project]( setup using cuda toolkit 11.7)
    • There are some errors installing cuda if you have the NVidia Game experience or Geforce drivers, just check which elements are failing to install and uncheck them from the installation (they fail because they are already installed), this was a slow process for me.
  3. Add the c compiler to the path adding cl.exe to the path
    • The official repo also talk about that here
  4. Install COLMAP per video instructions
  5. Install ImageMagik per video instructions
  6. Install FFMPEG per video instructions

Installation
...

  1. Install the repository using git shell. I tried using a client and the submodules were not correctly installed
# HTTPS
git clone https://github.com/graphdeco-inria/gaussian-splatting --recursive
  1. Install anaconda for managing python environments and createe a virtual environment
SET DISTUTILS_USE_SDK=1 # Windows only
conda env create --file environment.yml
conda activate gaussian_splatting
  1. I had to follow this instructions instead of the official ones to build the git submodules``
CONDA ENVIRONMENT  
I created the environment in conda using:  
`conda create -n gaussian_splatting python=3.7`  
activated the environment  
`conda activate gaussian_splatting`  
I then installed the dependencies step by step  
`conda install pytorch==1.13.1 torchvision==0.14.1 torchaudio==0.13.1 pytorch-cuda=11.7 -c pytorch -c nvidia`  
I changed directory to the directory where I cloned the repository with cd  
then I ran  
`pip install submodules/diff-gaussian-rasterization`  
and  
`pip install submodules/simple-knn`  
also I had to install manually the plyfile with  
`pip install plyfile`  
and tqdm with  
`pip install tqdm`

And that's it. I could keep on following the tutorial and avoid the error when running python train.py  
P.S. do not run the `conda env create --file environment.yml` or it breaks

Process images
...

  1. extract images from video
FFMPEG -i {path to video} -qscale:v 1 -qmin 1 -vf fps={frame extraction rate} %04d.jpg
  • I modified extraction rate to achieve around 200 images

estimating camera poses

  • Colmap for camera poses and intrinsic parameters
  • Steps are explained here: https://github.com/jonstephens85/gaussian-splatting-Windows#preparing-images-from-your-own-scenes
python convert.py -s <location> --skip_matching [--resize] #If not resizing, ImageMagick is not needed

Train the model to create GS files
...

  1. Per video and repository instructions run
python train.py -s <path to COLMAP or NeRF Synthetic dataset>
  • It takes around 20 mins to train

Rendering Splats
...

  1. The repository comes with a dektop viewer
./<SIBR install dir>/bin/SIBR_gaussianViewer_app -m <path to trained model>

Web renderer
...

I used the web viewer published on this repository

Usage
...

  1. Clone this repository and cd to its root folder
  2. mkdir data
  3. Extract some .splat files to data/ (e.g., example.splat)
  4. Start a local webserver python3 -m http.server 8000 --bind 127.0.0.1
  5. Open in the browser: http://127.0.0.1:8000/?url=http://127.0.0.1:8000/data/example.splat&z_is_up=true
    (or http://127.0.0.1:8000/?url=http://127.0.0.1:8000/data/example.splat if the model is oriented in a Y-is-up coordinate system)
